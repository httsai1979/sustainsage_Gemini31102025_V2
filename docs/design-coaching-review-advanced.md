# 進階全站自查（功能完整性 + 設計教練視角 20 問）

以下聚焦「功能可 100% 運作」與「卡片 / ICON / SEO / 社群 / AI 友好」的雙重標準。可作為衝刺檢核清單與改版計畫依據。

## 功能穩定度（10 問）
1. **導航/連結是否全站可點擊、無 404？** 逐頁跑鏈結測試並補足內鏈。 
2. **語系切換是否保持同頁內容？** hreflang、canonical、切換後的 URL 與文案是否一致。 
3. **表單是否可送出並回饋成功訊息？** 送出、錯誤、loading 狀態是否清楚。 
4. **CTA 是否有明顯焦點樣式與可點擊範圍？** 所有主要按鈕皆有 hover/focus ring。 
5. **資源下載/互動工具是否可開啟？** PDF、Notion、HTML 工具皆能成功開啟或下載。 
6. **圖片/ICON 是否正常載入與 lazy-load？** 無破圖、備援 alt 文案齊備。 
7. **卡片排版是否在手機仍可閱讀？** 單欄堆疊、內距一致、不被裁切。 
8. **深色模式或高對比模式表現？** 若有支援，文字/按鈕對比度 ≥ AA。 
9. **追蹤碼與事件是否正常送出？** GA4/GTM/像素的 CTA 與表單事件有回傳。 
10. **頁面速度與 CWV 指標是否過線？** LCP/CLS/FID (INP) 皆在建議值內，必要時延遲載入腳本。

## 內容與設計（10 問）
11. **Hero 是否一句話說痛點與解法？** 同時放置主/副 CTA。 
12. **卡片網格是否一致？** 服務、案例、資源都使用卡片 + ICON + 3–5 行摘要。 
13. **CTA 文案是否行為導向？** 例：「立即下載工作紙」「預約 20 分鐘諮詢」。 
14. **中文頁面是否仍殘留英文？** 找出預設英文化的 fallback，補齊在地化文案。 
15. **排版是否有足夠留白與層級？** H1–H3、粗體、列表、區塊留白 ≥ 24px。 
16. **信任背書是否可見？** 客戶 Logo、證言、認證徽章、媒體露出。 
17. **SEO/社群設定是否完整？** Title/Description、OG/Twitter Card、schema（FAQ/Article/ItemList）。 
18. **AI/推薦友好度？** 摘要精煉、列表化、卡片欄位結構化、關鍵字自然出現。 
19. **按鈕/卡片是否有微動畫與清晰對比？** hover/active/焦點具方向性（箭頭位移）。 
20. **內鏈與下一步是否清楚？** 每個模組都有「下一步」導向（案例→服務、資源→聯絡）。

## 改版與補強動作
- **中文在地化補齊**：針對所有 fallback 英文段落寫入 zh-TW/zh-CN 版本，含 CTA、流程、受眾卡片。
- **卡片 + ICON 一致化**：建立卡片樣式與行為（含微動畫、焦點樣式），套用於服務/案例/資源。
- **資源頁可用性**：逐項測試下載/互動工具連結，必要時提供備援連結或提示。
- **轉換與信任**：在 Hero、痛點段落、資源與 FAQ 下方加入 CTA；補充證言/徽章。
- **SEO/社群/AI**：檢查 canonical/hreflang、OG/Twitter、JSON-LD（FAQ/ItemList）；摘要與卡片文案 2–3 行、含關鍵字與利益點。
- **效能與可及性**：保留 lazy-load、圖片壓縮；按鈕對比、focus ring、字級 ≥16px；測試深色/高對比模式。
